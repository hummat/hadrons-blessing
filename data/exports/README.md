# Hadron's Blessing Exports

Checked-in JSON artifacts consumed by downstream projects (primarily BetterBots).

## Consumption model

**Primary:** Read files directly via filesystem (`../hadrons-blessing/data/exports/`).
**Secondary:** Regenerate via CLI commands in this repo.

Current BetterBots source of truth: `../BetterBots/scripts/mods/BetterBots/bot_profiles.lua` (`DEFAULT_PROFILE_TEMPLATES`).

## Artifacts

| File | Generated by | Consumer |
|------|-------------|----------|
| `bot-weapon-recommendations.json` | `npm run export:bot-weapons` or `npm run betterbots:sync` | BetterBots default profiles |

Related checked-in fixtures:
- `data/builds/bot/*.json` is generated by `npm run betterbots:sync` from the same BetterBots source and should stay in lockstep with the export.

## Staleness

Each artifact has a `generated_at` timestamp. Regenerate after:
- BetterBots profile changes in `bot_profiles.lua`
- Entity data changes affecting the export/build resolution
- Before any BetterBots release consuming the data

## Adding new exports

Only add exports justified by a concrete downstream use case. See spec:
`docs/superpowers/specs/2026-03-15-betterBots-integration-contract-design.md`
